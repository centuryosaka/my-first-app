## Git 運用ルール

### 基本方針

- コードを変更するたびに、必ず GitHub にプッシュすること
- 作業単位ごとにコミットを作成し、変更内容が明確になるようにすること

### コミットメッセージの書き方

- 日本語で記述してよい
- 変更の「なぜ」を中心に書く（何をしたかはコードから分かる）
- 例: `ログイン画面のバリデーション追加` / `パフォーマンス改善のためキャッシュを導入`

### ワークフロー

コードを変更したら、以下の手順を必ず実行すること：

```
git add <変更ファイル>
git commit -m "変更内容のメッセージ"
git push origin <ブランチ名>
```

### ブランチ戦略

- `main` ブランチ: 本番相当の安定版
- 機能追加・修正は feature ブランチを切って作業し、main にマージする
- 直接 `main` へのプッシュは避ける（緊急修正を除く）

### 禁止事項

- `git push --force` は原則禁止（やむを得ない場合はユーザーに確認する）
- `.env` や認証情報をコミットしない
- `--no-verify` でフックをスキップしない
